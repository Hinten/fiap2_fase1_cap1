Objetivo Geral da Atividade
Você é cientista de dados hospitalar na Fase 1 do projeto CardioIA (PBL – Project Based Learning).
Precisa buscar, organizar e preparar três tipos de dados cardiológicos que serão a base de todos os módulos de IA (Machine Learning, NLP, Visão Computacional, IoT etc.) nas próximas 7 fases.
Os dados devem ser:

Numéricos (pacientes cardíacos)
Textuais (médicos ou literários)
Visuais (imagens de exames)

Atividade Detalhada (3 partes obrigatórias)
Parte 1 – Dados Numéricos (IoT) – 3 pontos

Dataset mínimo 100 linhas com variáveis clínicas (idade, sexo, pressão arterial, colesterol, frequência cardíaca, histórico, sintomas etc.).
Criar repositório no GitHub.
No README.md: explicar origem (real ou simulado), link público (OneDrive/Google Drive/etc.) e destacar variáveis mais relevantes + justificativa clínica para IA.

Parte 2 – Dados Textuais (NLP) – 2 pontos

Baixar no mínimo 2 arquivos .txt (fontes: SciELO, BVS, SUS, Projeto Gutenberg).
Colocar em subpasta assets ou docs do GitHub.
No README: explicar como os textos podem ser usados por NLP (análise de sentimentos, extração de sintomas, classificação de tópicos) e justificar importância na saúde.

Parte 3 – Dados Visuais (Visão Computacional) – 2 pontos

Reunir no mínimo 100 imagens (.jpg ou .png) de exames (ECG, angiograma, raio-X torácico etc.).
Link público no README + justificativa clara de como a Visão Computacional pode analisar (detecção de padrões, anomalias etc.) e importância para IA na cardiologia.

Entregáveis Obrigatórios

Repositório GitHub com README.md detalhado (explicando todo o projeto + as 3 partes + fontes).
Subpasta com textos e imagens.
Links públicos (acessíveis para qualquer pessoa) para todos os dados (numéricos + imagens).
Tudo deve estar organizado, explicado e acessível para correção da FIAP.
